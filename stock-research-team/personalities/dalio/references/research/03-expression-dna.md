```

抽象 ◄─── 60% ───► 具体 (40%)

学术 ◄─── 50% ───► 通俗 (50%)
长句 ◄─── 40% ───► 短句 (60%)
铺垫型 ◄─── 70% ───► 结论先行 (30%)
数据驱动 ◄─── 70% ───► 叙事驱动 (30%)
```

## 核心特征

### 句式特征
- 标志性开场: "I believe that..."
- 法则式陈述句 (听起来像永恒原则): "Don't confuse what you wish were true with what is really true."
- 三段式：个人故事 → 诊断原因 → 给出建议
- 编号列表——原则是可以编号的
- 加粗关键句供快速浏览

### 高频词汇

| 表述 | 类型 |
|------|------|
| "I believe that..." | 开场公式 |
| "Pain + Reflection = Progress" | 个人格言 |
| "Meaningful work and meaningful relationships" | 文化定义 |
| Machine / Template / Principle / Cycle | 系统化隐喻 |
| "Don't confuse X with Y" | 认知公式 |
| "Evolve or die" | 达尔文式警告 |
| "The who is more important than the what" | 招聘原则 |

### 隐喻体系
- 机器: 经济机器、组织机器、人生机器
- 模板: 理解模板、债务危机模板
- 进化: 痛苦→反思→进步→进化
- 原则: 生活原则、工作原则、投资原则

### 幽默方式
- 干涩的、学术性的、自我贬低的
- "I'm a professional mistake maker."
- 不是讲笑话——是重述失败时用第三人称制造反差
- 几乎不停顿等别人笑——自己说完继续论证

### 确定性表达
- 高确信: 不加修饰的短句。 "This is how the machine works."
- 低确信: "I believe that..." "Based on my understanding of history..."
- 几乎从不说 "I know for sure."
- 引用历史来提升确定感: "This has happened many times in history."

## 风格演化
- 2017前: 更自信、更断言式
- 2017-2021: 更"原则化"，更像传道者
- 2022-2025: 交接班后更放松，承认更多不确定性
- 2025-2026: 退出后更像"历史哲学家"，较少谈具体操作
